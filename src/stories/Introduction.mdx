# Skyfall Aegis

Aegis is the HealthTech UI kit in the Skyfall family of design systems. It provides a comprehensive set of React components purpose-built for healthcare SaaS products — from patient-facing portals to provider dashboards and clinical workflows.

## What Aegis is for

Aegis is designed to accelerate product design and front-end development for healthcare applications, including:

- **Patient and provider interfaces** — scheduling, messaging, records access
- **Care management workflows** — care plans, referrals, task tracking
- **Clinical dashboards** — lab results, vitals, medication management
- **Administrative tools** — insurance verification, billing, reporting
- **Data-rich experiences** — tables, charts, timelines, and status-heavy layouts

## Who Aegis is for

Aegis is built for product engineers, designers, and teams building digital health products that need to be trustworthy, accessible, and production-ready from day one.

## System structure

Aegis is organized into two layers:

### Components

The shared core system — foundational building blocks used across all Skyfall UI kits. This includes inputs, data display, feedback, surfaces, navigation, layout primitives, and utilities. These components follow atomic design principles and compose predictably.

### Aegis X

The HealthTech-specific extension layer. This includes healthcare domain components (PatientCard, ClinicalBanner, MedicationRow, and others) and a Recharts-based data visualization system built for clinical and operational dashboards.

## Design principles

Aegis components are built around five qualities:

- **Trust** — consistent visual language and predictable behavior that inspires confidence in clinical contexts
- **Clarity** — clean information hierarchy that supports data-dense interfaces without visual noise
- **Accessibility** — WCAG 2.1 compliant, keyboard-navigable, screen-reader-friendly by default
- **Composability** — atomic primitives that compose into complex healthcare patterns
- **Precision** — type-safe APIs, semantic tokens, and well-defined component boundaries
